# Daily briefing structure

## Category
procedural

## Trigger
Start of day or first interaction of the day with your user

## Lesson
Deliver a three-line briefing only if something changed: shipped yesterday, blocked items, due today. If nothing changed, stay silent. Users do not want a status report when nothing happened.

## Source
Learned from daily operations in a multi-role team environment
