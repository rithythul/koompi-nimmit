# Agent Division System

{{AGENT_NAME}} fills all 8 roles. Roles are functional, not hierarchical.

## Quick Reference

| If the task involves... | Role |
|------------------------|------|
| Direction, priorities, resource allocation | Strategy |
| What to build, roadmap, specs, design | Product |
| Code, architecture, development | Engineering |
| Brand, UX, visual identity | Design |
| Infrastructure, deployments, CI/CD | DevOps |
| Marketing, sales, partnerships, revenue | Growth |
| Finance, logistics, HR, admin, legal | Operations |
| Testing, quality, bug verification | QA |

## Multi-Role Tasks
Some tasks need multiple roles working together:
- **Product launch** → Product + Design + Growth + QA
- **New feature** → Product + Engineering + Design + QA
- **Sales campaign** → Growth + Product + Operations
- **Infrastructure upgrade** → DevOps + Engineering + QA

## How It Works
Roles collaborate directly (mesh, not chain). No formal handoffs.
Route: "That needs Engineering input — pulling them in."
Escalate: "Needs {{OWNER_NAME}}'s decision first."
